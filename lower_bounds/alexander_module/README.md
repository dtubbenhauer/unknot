# Complete finite-field Alexander-module calculation

For a Seifert matrix `V`, finite field `F`, and nonzero `alpha` in `F`,

```text
nullity_F(alpha V - V^T) <= e(K) <= u(K),
```

where `e(K)` is the Nakanishi index.

The calculation here is exhaustive. Let `M(t)=tV-V^T` be an `m`-square
presentation and let `E_r(M)` be generated by its `(m-r+1)`-minors. The largest
nullity over all finite fields is

```text
beta(K) = max { r : E_r(M) is not the unit ideal }.
```

Thus no unbounded enumeration of fields is required. The exact scan uses unit
pivots to reduce each presentation and then tests these elementary ideals over
`Z[t,t^-1]`.

## Bounds and the Nakanishi index

If a nullity-`r` witness is defined over `F_(p^ell)`, it can be chosen with
`ell <= m-r+1`. For a genus-`g` matrix of an `n`-crossing knot this gives
`ell <= 2g-r+1 <= n-r+1`.

For a fixed integral presentation, the finite-field index agrees with the
characteristic-zero maximum for all but finitely many effectively computable
primes. There is no prime bound depending only on the genus, although a coarse
crossing-number bound follows from a diagram-derived Alexander presentation.

Finally,

```text
beta(K) <= e(K) <= beta(K)+1.
```

The possible gap occurs already for `9_38`, so finite-field specializations do
not by themselves compute the exact Nakanishi index in every case.

## Reproduction

The third-party KnotInfo workbook is not redistributed. With the matching file
recorded in `data/workbook_provenance.json`, run from the repository root:

```bash
python3 code/complete_alexander_scan.py "$WORKBOOK" \
  --output build/complete_alexander_scan.json
python3 code/verify_complete_alexander_witnesses.py "$WORKBOOK" \
  --scan build/complete_alexander_scan.json \
  --output build/direct_rank_verification.json
```

Before using a Seifert matrix, the restricted scan also checks

```text
|det(V - V^T)| = 1,
|det(V + V^T)| = det(K).
```
